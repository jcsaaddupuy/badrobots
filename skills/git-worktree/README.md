# Git Worktree Skill

## Description
Comprehensive guide for using `git worktree` to manage multiple working directories for parallel development, code reviews, hotfixes, and testing.

## What This Skill Covers

- Complete `git worktree` command reference (add, list, remove, move, lock, prune, repair)
- Common workflows (hotfixes, code reviews, parallel development, testing)
- Best practices for naming, placement, cleanup, and branch management
- Troubleshooting guide for common issues
- Scripting and automation helpers
- Performance considerations and when to use worktrees

## When to Use This Skill

Use this skill when you need to:
- Work on multiple branches simultaneously without constant switching
- Review PRs without interrupting your current work
- Handle urgent hotfixes while developing features
- Test multiple branches in parallel
- Run long-running builds/tests on different branches
- Understand git worktree best practices and workflows
